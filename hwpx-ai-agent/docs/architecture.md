# HWPX AI Agent Architecture

## 목표

HWPX AI Agent는 외부 AI API 없이 내부 PC 또는 내부망에서만 동작하는 문서 업무 자동화 시스템이다. 문서 원문, 로그, 작업 계획, 수정본은 모두 로컬 파일시스템과 SQLite에 저장한다.

## 핵심 흐름

1. 사용자가 HWPX 파일을 업로드한다.
2. 백엔드는 원본을 `data/originals`에 저장하고 작업 사본을 `data/workspaces`에 안전하게 압축 해제한다.
3. HWPX 파서는 XML을 내부 문서 모델로 변환한다.
4. 사용자의 자연어 요청은 에이전트 플래너가 JSON 작업 계획으로 변환한다.
5. Pydantic 스키마가 계획과 도구 인자를 검증한다.
6. 읽기/검색/검증 작업은 즉시 실행할 수 있다.
7. 문서 수정 작업은 `pending_approval` 상태로 저장된다.
8. 승인된 계획만 검증된 Python 편집 도구가 실행한다.
9. 수정된 작업 사본을 새 HWPX로 패키징하고 XML/ZIP 검증을 수행한다.
10. 원본과 수정본의 XML diff, 실행 결과, 감사 로그를 저장한다.

작성형 요청은 별도 계획 경로를 사용한다. 먼저 `detect_template_fields`가 `{{제목}}`, `[제목]`, `____`, `○○○` 같은 자리표시자를 찾는다. 자리표시자가 있으면 Ollama가 필드별 값을 생성하고, 백엔드는 `fill_template_fields` 도구로 승인 후 정확한 대상 문자열만 치환한다. 자리표시자가 없으면 Ollama가 초안 문단 배열을 생성하고, 백엔드는 `append_paragraphs` 도구로 승인 후 HWPX 본문 끝에 문단을 추가한다. 이때도 LLM은 XML을 직접 수정하지 않는다.

## 보안 경계

- LLM은 XML, 파일 경로, Python 코드, 셸 명령을 직접 실행하지 않는다.
- 모든 수정은 내부 ID와 검증된 도구 함수만 사용한다.
- ZIP Slip 방지를 위해 모든 압축 멤버 경로를 검사한다.
- 압축 파일 개수와 해제 후 총 용량을 제한한다.
- XML DTD와 엔티티 선언은 거부한다.
- 원본 파일은 수정하지 않는다.
- 삭제와 덮어쓰기는 기본 기능에 포함하지 않는다.

## 백엔드 모듈

- `app/hwpx`: HWPX 패키지, 파서, 편집기, 검증기, diff
- `app/security`: 파일 경로와 ZIP 안전성 검사
- `app/llm`: Ollama 클라이언트, JSON 복구 파서, 계획 스키마
- `app/agents`: 모델 선택, 프롬프트, 계획 생성, 오케스트레이션
- `app/tools`: 승인된 Python 도구 레지스트리
- `app/services`: 문서 저장, 감사 로그, 계획 실행
- `app/db`: SQLite 스키마와 저장소
- `app/api`: FastAPI 라우터

## 프론트엔드

React/Vite 앱은 한국어 UI로 문서 업로드, 목록, 구조/표 보기, AI 채팅, 계획 승인, diff, 검증, 모델 설정 화면을 제공한다. MVP에서는 단일 사용자 로컬 워크플로를 전제로 한다.

## MVP 한계

이미지, 수식, 차트, 도형은 읽기 전용으로 보존한다. 한컴오피스 렌더링, 전자결재 연동, 다중 사용자 권한 관리는 확장 인터페이스 이후 단계에서 구현한다.
